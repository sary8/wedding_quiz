# 2026-07-16 Lighthouse 100点対応

`/play`(参加者入口)の Lighthouse 測定(本番ビルド + モバイル条件)で Performance 69 / Accessibility 100 / Best Practices 96 / SEO 82。全カテゴリ 100 点にするための修正記録。

**結果: Performance 100 / Accessibility 100 / Best Practices 100 / SEO 100(3回測定で安定、FCP 1.4s / LCP 1.7s / TBT 0ms / CLS 0)**

## P-1. Google Fonts が render-blocking で FCP/LCP 5.0s(Performance 69)

- `index.html` が fonts.googleapis.com の CSS(122KB)を同期ロード。render-blocking で推定 3.6s の遅延。
- 読み込んだ CSS の 115KB(94%)が未使用。3書体 × 計10ウェイトを全部リクエストしているため。
  - Cormorant Infant 300/400/500/600/700 → 実使用は 400 のみ(`.font-serif-wedding` にウェイト併用クラスなし)
  - Noto Sans JP 300/400/500/700 → 300 は未使用。`font-semibold(600)` / `font-extrabold(800)` は未ロードのため従来から 700 で合成描画
  - Great Vibes 400(唯一のウェイト)
- 対応(最終形):
  - **日本語 Web フォントは廃止**しシステムフォントスタックへ(Android=Noto Sans CJK, iOS/macOS=ヒラギノ, Windows=游ゴシック)。@fontsource/noto-sans-jp も試したが unicode-range 分割の @font-face 360宣言で CSS が +300KB(gzip +100KB)になり本末転倒だった。
  - **装飾書体(Great Vibes / Cormorant Infant)は latin サブセットのみ** `public/fonts/` からセルフホスト。表示テキストが英字のみのため十分。
  - `index.html` で `<link rel="preload" as="font">` + インライン @font-face 宣言。CSS→フォントの直列ロードを解消し、LCP 要素(h1)のスワップを前倒し。
- 副次効果: 外部オリジン依存が消え、式場の回線で Google Fonts が不通でもデザインが崩れない。

## P-2. favicon.ico が 404 でコンソールエラー(Best Practices 96)

- icon link が未宣言のためブラウザが `/favicon.ico` を自動リクエストし 404。
- 対応: ウェディングテーマ(サージュグリーン + ゴールドの指輪 + ダイヤ)の SVG favicon を作成し、`favicon.svg` / `favicon.ico`(16/32/48px PNG格納、3.8KB)/ `apple-touch-icon.png` を `public/` に配置、`index.html` にリンク追加。`theme-color` も追加。

## P-3. meta description なし・robots.txt が無効(SEO 82)

- meta description が未定義。`/robots.txt` が SPA フォールバックで HTML を返し 18 エラー。
- 対応: `index.html` に description を追加、`public/robots.txt`(全許可)を配置。

## P-4. Vite がフォントを CSS に base64 インライン化し CSS が肥大(追加検出)

- `build.assetsInlineLimit` デフォルト(4KB)により、4KB 未満のフォントチャンクが data URI として CSS に埋め込まれ、render-blocking CSS が 406KB(gzip 124KB)に膨張していた。
- 対応: `assetsInlineLimit: 0` を設定。

## P-5. manualChunks の設定不備でチャンク分割が機能していなかった(追加検出)

- `"vendor-react": ["react", "react-dom", ...]` はパッケージエントリしか捕捉せず、本体を持つ `react-dom/client`(525KB, gzip 53KB)が index チャンクに漏れていた。`"react-dom/client"` を明示して解消。
- `"vendor-motion": ["framer-motion"]` は利用者が全員 lazy ページにもかかわらず、手動チャンク化により index からの静的依存となり `<link rel="modulepreload">` で初期ロードに 40KB (gzip) 注入されていた。手動チャンク指定を削除し自動分割に変更(遅延側に残る)。

## P-6. autoFocus の同期フォーカスで forced reflow 130ms(追加検出)

- JoinPage の `<Input autoFocus>` が React commit 直後に同期 focus() を呼び、スタイル未確定状態で強制レイアウト(130ms、4x CPU スロットルで TBT 160ms)。
- 対応: `useEffect` + `requestAnimationFrame` で初回ペイント後にフォーカス。

## P-7. SPA の白画面待ちで FCP/LCP が JS 実行に律速(追加検出)

- CSR のため FCP/LCP が「JS 92KB(gzip) の DL + 実行」に律速され、シミュレーション値 1.7s から下がらなかった。
- 対応: `index.html` の `#root` 内に **JoinPage と同一見た目の静的アプリシェル**を配置。React 起動前(実測 FCP 352ms)にタイトル・カードが描画される。
  - 構造・クラスは JoinPage.tsx と一致させること(ズレると置換時に CLS が発生)。
  - エントランスアニメーション(animate-fade-up 等)は**シェル側だけ**に付ける。React 側に付けると置換時に再生し直されてチラつく。
  - `/host/*` 等の他ルートでも一瞬表示されるが、ブランドスプラッシュとして許容。

## 対象外

- agentic-browsing カテゴリ(llms.txt)は Lighthouse 13 の実験的項目のため対応しない。

## 回帰防止

- `src/frontend/src/indexHtml.test.ts`: meta description / favicon リンク / 外部フォント不使用 / preload + @font-face / 静的シェルの存在を静的検証。
