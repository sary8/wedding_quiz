# 2026-06-28 チーム戦の仕様変更: A〜Zチーム・幹事が個数設定・常に個人戦＋裏でチーム戦

## 背景 / 目的
結婚式二次会の受付で各ゲストにチームを割り振る運用に合わせ、チーム戦の仕組みを変更する。
- 受付で配られたチーム（A〜Z）を、参加登録画面で**本人が選ぶ**
- 幹事は**チーム数（A〜Zのうち何個）**を設定するだけ（カスタムチーム名は廃止）
- **ON/OFFトグルを廃止**し、常に「個人戦（表）＋チーム戦（裏で集計）」が動く

## 確定要件（ユーザー回答）
1. 振り分け = 本人がA〜Zから選択（物理的な割当はアプリ外＝会場受付）
2. チーム数 = 幹事が個数を設定（2〜26、A〜（N文字目））
3. ON/OFFトグル = 廃止して常に両方

## 変更方針
### バックエンド（src/backend）
- `routes/quiz.ts`
  - `POST "/"`（クイズ作成）: 作成時に `team_mode=true` をセットし、既定チーム（A〜D の4チーム）を自動作成 → どのクイズも必ずチーム戦が成立。
  - `PUT "/:id/teams"`: 上限 10 → 26 に拡大。チーム名1文字(A〜Z)許容。保存時に `team_mode=true` も更新。
  - `PUT "/:id/team-mode"`: UIからは未使用化（後方互換でエンドポイントは残置）。
- `services/quizService.ts` `joinRoom`: 既存の「team_mode時 teamId必須」バリデーションがそのまま機能（変更最小）。
- スキーマ/マイグレーション変更は不要（`team_mode` はアプリ側で常にtrueをセット）。

### フロントエンド（src/frontend）
- `components/setup/QuizConfigTab.tsx`: トグル＋チーム名入力リストを撤去 → 「チーム数(2〜26)」設定に置換。保存時に `[{name:"A"},…]` を生成して `setTeams`。`String.fromCharCode(65+i)`。
- `pages/participant/ProfilePage.tsx`: チーム選択を常時表示。A〜Z レターボタンで選択（未選択は参加不可）。
- ランキング/最終発表/参加者画面: 既に `teamRankings` を扱えるため常時表示になる以外の改修は基本不要。teamName=レター表示の見た目だけ調整。

## テスト
- backend: チーム最大26件・作成時の既定チーム生成・参加時 teamId 必須。
- frontend: 旧トグル/カスタム名に依存するテストがあれば個数設定UIへ更新。

## 検証
- `src/backend` / `src/frontend` の build と既存テスト（vitest / e2e）が通ること。
- 手動: クイズ作成→チーム数設定→ロビー→A〜Zで参加→個人＆チームランキング→最終発表。
