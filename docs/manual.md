## 1. システム構成

```
参加者スマホ（最大100人）
      ↕ Socket.io / HTTP
    バックエンド API（Node.js + Hono）
      ↕ SQLite
    フロントエンド（React + Vite）
      ↙ ホストPC    ↘ プロジェクター
```

| 役割 | URL（本番） | URL（ローカル開発） |
|------|-------------|-------------------|
| フロントエンド | Azure Static Web Apps の URL | http://localhost:5174 |
| バックエンド API | Azure App Service の URL | http://localhost:3001 |

## 2. 事前準備

### 2-1. 必要なもの

- **ホスト用PC**（司会者が操作）— ブラウザが動けば何でも可
- **プロジェクター + HDMI ケーブル**（または大型モニター）
- **Wi-Fi ルーター**（参加者全員が接続できるもの）
- **参加者スマホ**（QRコード読み取り可能なブラウザ搭載端末）

### 2-2. ネットワーク要件

- ホストPC・プロジェクターPC・参加者スマホが **同一ネットワーク**（またはインターネット経由）で接続できること
- バックエンドのポート **3001**（開発時）または Azure URL（本番）が全端末からアクセス可能なこと

### 2-3. ブラウザ推奨

| 端末 | 推奨ブラウザ |
|------|-------------|
| ホストPC | Chrome 最新版 |
| プロジェクター | Chrome 最新版 |
| 参加者スマホ | Chrome / Safari 最新版 |

## 3. クイズ作成（SetupPage）

### アクセス方法

```
http://<サーバーURL>/host/setup
```

### 手順

#### Step 1: クイズを作成する

1. 「タイトル」欄にクイズ名を入力（例: `新郎新婦クイズ`）
2. 「クイズを作成」ボタンをクリック
3. 作成したクイズが画面下部の一覧に表示される

#### Step 2: 問題を追加する

各問題について以下の項目を入力：

| 項目 | 説明 | 必須 |
|------|------|------|
| 問題文 | 参加者に表示する質問文 | ◎ |
| 選択肢1〜4 | 4つの回答候補 | ◎ |
| 正解番号 | 1〜4のいずれか | ◎ |
| 制限時間 | 10〜120秒（デフォルト30秒） | ◎ |
| 画像 | 問題に表示する画像（任意） | — |

**画像添付の手順：**
1. 「画像を選択」ボタンをクリックしてファイルを選択（JPG・PNG・GIF・WebP、最大5MB）
2. サムネイルが即座に表示される（バックグラウンドでアップロード中）
3. アップロード完了後、問題文の横に「✓ 完了」バッジが表示される
4. 「追加」ボタンが活性化したら問題を保存できる
5. 削除したい場合は「×」ボタンをクリック

#### Step 3: ロビーを開く

1. 問題をすべて追加したら「ロビーを開く」ボタンをクリック
2. 自動的にホスト操作画面（`/host/:roomCode`）へ遷移する

> **注意**: ロビーを開く前に Host Secret Key が localStorage に保存されます。
> 同じブラウザからのみホスト操作が可能です。別ブラウザで操作する場合は URL の `?key=` パラメータを手動でコピーしてください。

## 4. 当日の進行手順

### 4-1. サーバー起動

**ローカル開発環境の場合：**

ターミナルを2つ開いて、それぞれ実行：

```bash
# ターミナル1：バックエンド
cd src/backend
npm run dev
# → ポート3001で起動

# ターミナル2：フロントエンド
cd src/frontend
npm run dev
# → http://localhost:5174 で起動
```

**本番環境（Azure）の場合：**

Azure Portal でバックエンドの App Service を起動済みであることを確認。

---

### 4-2. ロビー開設〜参加者受付

1. ホストPC でブラウザを開き、`/host/setup` へアクセス
2. クイズを選択または新規作成して「ロビーを開く」
3. ホスト操作画面に **QRコード** と **ルームコード（6文字）** が表示される
4. 参加者にQRコードを読み取らせる（または `/play` のURLとルームコードを口頭で案内）
5. 参加者一覧にニックネームが表示されていくのを確認

---

### 4-3. プロジェクター接続

プロジェクターには **操作ボタンなしの表示専用画面** を使います。

1. プロジェクターに接続したPCのブラウザで以下のURLを開く：

```
http://<サーバーURL>/host/<ルームコード>/screen
```

例: `http://localhost:5174/host/ABCDEF/screen`

> **ヒント**: ホスト操作画面のロビー画面の下部に「プロジェクター用URL」が表示されているのでコピー＆ペーストしてください。

2. プロジェクター画面はホスト操作画面と **自動同期** します
3. ホストが操作するたびに画面が切り替わります

---

### 4-4. ゲーム開始〜問題進行

#### ゲーム開始

1. 参加者が揃ったら「ゲーム開始」ボタンをクリック
2. 全員の画面が「回答待ち」に切り替わる

#### 問題の進め方

各問題で以下の流れを繰り返す：

```
[次の問題] → 問題表示・カウントダウン開始
    ↓ (時間内に参加者が回答)
[回答を締め切る] または 時間切れで自動締め切り
    ↓
正解と回答分布を確認
    ↓
[ランキングを表示] → 上位10名のバーアニメーション
    ↓
[次の問題] or [最終結果発表]（最終問題の場合）
```

#### 操作ボタン一覧（問題フェーズ）

| ボタン | タイミング | 説明 |
|--------|----------|------|
| 次の問題 | 準備完了時 | 次の問題を全員に配信 |
| 回答を締め切る | 問題表示中 | 手動で早期締め切り |
| ランキングを表示 | 結果確認後 | 上位10名のバーグラフを表示 |
| 最終結果発表 | 全問終了後 | オールスター感謝祭式の発表 |

---

### 4-5. ランキング〜最終発表

#### ランキング画面

- 上位10名が左からバーが伸びるアニメーションで表示
- スコアが高い順に表示
- 参加者のスマホにも同じランキングが表示される

#### 最終結果発表

- 全問題終了後「最終結果発表」ボタンをクリック
- 10位から1位へカウントダウン形式で発表（オールスター感謝祭スタイル）
- 自動進行（操作不要）
- 参加者スマホには自分の最終順位・正答数・平均回答速度が表示される

## 5. 参加者の操作手順

### Step 1: 参加する

1. QRコードをスマホのカメラで読み取る（または `/play` のURLをブラウザで開く）
2. 表示された画面でルームコード（6文字）を入力して「参加する」

### Step 2: プロフィール設定

1. ニックネームを入力（最大10文字）
2. 任意でカメラで自撮り撮影（スキップ可能）
3. 「参加する」ボタンをタップ

### Step 3: 待機〜ゲーム開始

- ホストがゲームを開始するまで待機画面が表示される
- ゲーム開始と同時に自動で回答画面に切り替わる

### Step 4: 回答する

- 問題文が表示されたら4色のボタンから答えを選ぶ
- 選択後はボタンが押せなくなる（変更不可）
- 制限時間内に回答しないと0点

### Step 5: 結果確認

- 正解・不正解と獲得ポイントが表示される
- ポイントは **早押しボーナス付き**（速く答えるほど高得点）
- 現在順位も確認できる

### 再接続について

- ブラウザを閉じてしまっても `localStorage` のトークンで自動復帰
- 同じURLに再アクセスすれば続きからゲームに参加できる

## 6. 各画面の詳細説明

### 6-1. ホスト画面一覧

#### `/host/setup` — 問題作成・管理画面

- クイズの新規作成（タイトル入力）
- 既存クイズの選択・削除
- 問題の追加（問題文・選択肢4択・正解・制限時間・画像）
- 問題の削除
- ロビーを開くボタン

#### `/host/:roomCode?key=HOST_SECRET&quizId=QUIZ_ID` — ホスト操作画面

フェーズごとの表示と操作：

| フェーズ | 表示内容 | 操作ボタン |
|----------|----------|-----------|
| lobby | QRコード・ルームコード・参加者一覧 | ゲーム開始 |
| question | 問題文・選択肢・カウントダウン・回答数 | 回答を締め切る |
| results | 回答分布グラフ・正解表示 | ランキングを表示 / 次の問題 |
| ranking | Top10 バーアニメーション | 次の問題 / 最終結果発表 |
| final | カウントダウン発表 | なし（自動進行） |

#### `/host/:roomCode/screen` — プロジェクター専用表示画面

- ホスト操作画面と完全同期（読み取り専用）
- 操作ボタンは一切なし
- ゲーム開始後にアクセスしても既存状態を取得して同期

---

### 6-2. 参加者画面一覧

#### `/play` — ルームコード入力画面

- 6文字のルームコードを入力して参加

#### `/play/:roomCode` — 参加フロー画面

フェーズごとの表示：

| フェーズ | 表示内容 |
|----------|----------|
| profile | ニックネーム入力 + 自撮り撮影（任意） |
| waiting | ゲーム開始待機画面 |
| answer | 4色ボタンで回答・制限時間カウントダウン |
| result | 正解/不正解・獲得ポイント・現在順位 |
| ranking | ランキング表示（ホストと連動） |
| final | 自分の最終順位・正答数・平均回答速度 |

## 7. トラブルシューティング

### Q: 参加者がQRコードを読み取れない

**A:** 以下を確認してください
- プロジェクターへの投影が見づらい場合は、ホストPCの画面を直接見せるか、URLとルームコードを口頭で案内する
- QRコードに十分な余白があるか確認（プロジェクターの端が切れていないか）

---

### Q: 参加者の画面が「接続中…」から進まない

**A:**
1. 参加者のスマホが Wi-Fi に接続されているか確認
2. バックエンドサーバーが起動しているか確認（`http://<バックエンドURL>/api/quizzes` でレスポンスがあるか）
3. スマホのブラウザをリロードする

---

### Q: ホスト操作画面に「接続エラー」が表示される

**A:**
1. バックエンドサーバーが起動しているか確認
2. URLの `key=` パラメータが正しいか確認（SetupPage から遷移した場合は自動設定される）
3. ブラウザの localStorage をクリアして再度 SetupPage からやり直す

---

### Q: プロジェクター画面がホスト画面と同期しない

**A:**
1. プロジェクターPCのブラウザで `/host/:roomCode/screen` のURLが正しいか確認
2. ブラウザをリロードする
3. それでも解決しない場合は、ゲーム開始後でも `/screen` URLに再アクセスすれば現在の状態が取得される

---

### Q: 回答が送信できない（ボタンが反応しない）

**A:**
- ホストが問題を締め切った後は回答できません（正常な動作）
- 制限時間が切れた後も回答できません（正常な動作）
- 通信切れの場合はリロードして再接続してください

---

### Q: 参加者がブラウザを閉じてしまった

**A:**
- 同じURLに再アクセスすると `localStorage` のトークンで自動復帰します
- ゲーム途中でも再接続後、次の問題から参加できます

---

### Q: 画像がアップロードできない / 表示されない

**A:**
- ファイルサイズが5MB以上の場合はアップロードできません
- 対応フォーマット：JPG・JPEG・PNG・GIF・WebP のみ
- バックエンドの `uploads/` ディレクトリに書き込み権限があるか確認

---

### Q: 途中でサーバーが落ちてしまった

**A:**
1. バックエンドを再起動する
2. **ゲームの状態はメモリ上に保持されているため、再起動するとリセットされます**
3. ゲームを最初からやり直す必要があります
4. 本番では Azure App Service の「Always On」設定を有効にしておくことを推奨します

## 8. Azure デプロイ手順

### フロントエンド（Azure Static Web Apps）

1. Azure Portal で「Static Web App」を新規作成
2. GitHub リポジトリと連携（または手動デプロイ）
3. ビルド設定：
   - アプリの場所: `src/frontend`
   - ビルドコマンド: `npm run build`
   - 出力フォルダ: `dist`
4. 環境変数 `VITE_API_URL` にバックエンドのURLを設定

### バックエンド（Azure App Service B1 Linux）

1. Azure Portal で「App Service」を新規作成
   - ランタイム: Node.js 20 LTS
   - プラン: Basic B1（または S1）
   - OS: Linux
2. デプロイメント設定（GitHub Actions または ZIP デプロイ）
3. 環境変数を設定：
   - `HOST_SECRET`: ホスト認証用シークレット
   - `NODE_ENV`: `production`
   - `PORT`: `8080`（App Service のデフォルト）
4. スタートアップコマンド: `node dist/index.js`
5. **CORS 設定**: フロントエンドの Static Web Apps URL を許可

### SQLite データの扱い

- App Service の `/home` 以下は永続化されます（再起動後も保持）
- データディレクトリを `/home/data` に設定することを推奨
- 大規模な本番利用には Azure SQL や Turso（SQLite as a Service）への移行を検討

## 9. コスト概算

| サービス | プラン | 月額（USD） | 月額（JPY 目安） |
|----------|--------|-------------|----------------|
| Azure App Service B1 Linux | Basic B1（1コア・1.75GB RAM） | ~$13.14 | ~¥2,000 |
| Azure Static Web Apps | Free Tier | $0 | ¥0 |
| **合計** | | **~$13/月** | **~¥2,000/月** |

### 結婚式当日のみ使う場合

- App Service は **停止** にすると課金が止まります（コンピューティングのみ。ストレージは微課金）
- イベント前日に起動 → 翌日終了で実質 **2〜3日分** ≒ **¥200〜300** 程度
- Static Web Apps Free Tier は常時無料

### 費用を節約する方法

- ゲーム当日だけ App Service を起動する（平常時は停止）
- 参加者100人・1時間のゲームなら Free Tier の帯域制限（100GB/月）は超過しない
- 自撮り写真を多用する場合はストレージも確認（uploads/ ディレクトリ）
