# Wedding Quiz App 機能仕様書

> バージョン: 1.2.0
> 最終更新: 2026-02-28
> ステータス: 確定

---

## 概要

結婚式の二次会用 Kahoot! 形式リアルタイムクイズアプリ。
ホスト（PC/プロジェクター）から問題を出し、参加者約100人がスマホから回答する。

---

## F001: クイズ管理

### 概要
ホストがクイズの作成・編集・削除・問題管理を行う。

### 受入基準
- [ ] クイズの新規作成（タイトル入力）
- [ ] クイズタイトルの編集（インライン編集）
- [ ] クイズの削除（cascade で問題・参加者・回答も削除）
- [ ] 問題の追加（問題文・選択肢4択・正解番号・制限時間5-120秒・画像任意）
- [ ] 問題の編集（全フィールド更新可能）
- [ ] 問題の削除
- [ ] 問題の並べ替え（D&D + ↑↓ボタン）
- [ ] 問題文: 最大500文字、選択肢: 最大200文字
- [ ] 画像: JPG/PNG/GIF/WebP、最大5MB、magic byte検証
- [ ] mediaType: "image" のみ（動画UIは実装しない）

### UI
- SetupPage ダッシュボードハブ（作成・一覧・履歴・ギャラリー・ライブラリ）
- QuestionManagementTab（@dnd-kit D&D + 手動↑↓ボタン）

## F004: ゲーム進行

### 概要
ホストが問題を配信し、参加者がリアルタイムで回答する。

### ゲームフロー
```
lobby → countdown(5秒) → question → results → ranking → [次の問題 or 最終発表]
                                                              ↓
                                                     final → group → closed
```

### 受入基準
- [ ] 5秒カウントダウン後に最初の問題を自動配信
- [ ] サーバーサイドタイマー（hrtime高精度、クライアント改ざん不可）
- [ ] 4色回答ボタン（赤・青・緑・黄）
- [ ] 回答二重送信防止（UNIQUE制約 question_id + participant_id）
- [ ] 手動締め切り or タイマー自動締め切り
- [ ] 回答分布バーチャート（結果フェーズ）
- [ ] 参加者に正解テキスト表示
- [ ] ライブ回答数表示（参加者画面にも）
- [ ] タイマー値クランプ（Math.max(0, remaining)）

### スコアリング
- 正解: 500〜1000点（速度ボーナス、早いほど高得点）
- 不正解: 0点
- 計算式: `baseScore * (1 - responseTime / timeLimit * 0.5)`

### Socket.io
- `startGame` → `gameStarted`
- `nextQuestion` → `questionStarted`
- `timeUpdate` — 毎秒残り時間
- `answerCountUpdate` — リアルタイム回答数
- `submitAnswer` — 回答送信
- `closeQuestion` → `questionClosed` → `questionResult`

- [ ] 集合写真フェーズ: 浮遊アバターアニメーション（スクリーンセーバー風）
- [ ] prefers-reduced-motion時は静的グリッド表示

### Socket.io
- `endGame` → `gameEnded`

## F017: 画像選択肢

### 概要
問題の選択肢にテキストだけでなく画像を使えるようにする。

### 受入基準
- [ ] 問題ごとに「テキスト選択肢」か「画像選択肢」を切り替え（混在なし）
- [ ] 画像選択肢にはオプションでテキストラベルを付けられる（最大200文字）
- [ ] 既存の問題レベル画像（`media_url`）と画像選択肢は併用可能
- [ ] 既存の問題はすべて `choice_type = "text"` として後方互換
- [ ] テンプレート（問題バンク）でも画像選択肢をサポート
- [ ] テンプレートからインポート時に画像URLも保持

### DB変更
`questions` と `question_bank` テーブルに以下のカラムを追加:
- `choice_type` TEXT NOT NULL DEFAULT 'text' — "text" | "image"
- `choice1_image_url` TEXT — nullable
- `choice2_image_url` TEXT — nullable
- `choice3_image_url` TEXT — nullable
- `choice4_image_url` TEXT — nullable

### バリデーション
- `choice_type === "text"`: 従来通り choice1-4 テキスト必須
- `choice_type === "image"`: choice1-4 テキストは任意（ラベル）、代わりに image_url 4つ必須
- 画像フォーマット: JPG/PNG/GIF/WebP、最大5MB（問題画像と同一制約）
- magic byte検証あり

### 画像ガイドライン
- 推奨アスペクト比: 正方形（1:1）または 4:3
- 推奨解像度: 最低500×500px
- ホスト画面では `object-cover` でクロップ表示されるため、被写体を中央に配置推奨
- 参加者画面では `object-cover` でボタン内に収まる表示

### UI

#### 問題作成（QuestionInlineForm）
- テキスト/画像の切替トグルボタン
- 画像モード時: 各選択肢に画像アップロード UI + オプションのテキストラベル入力
- アップロード中は「アップロード中…」表示、完了でサムネイル表示
- 切替時のデータ保持: テキスト↔画像を切り替えても入力済みデータは保持される（元に戻せば復元可能）
- 保存条件: 画像モードでは4つの画像すべてアップロード完了が必須

#### ホスト/プロジェクター画面（QuestionPage）
- 2×2グリッドに画像サムネイル（112-144px）+ テキストラベル
- 画像ありの場合フォントサイズを縮小して画像スペースを確保
- `object-cover` + 角丸で統一表示

#### 参加者画面（AnswerPage / ChoiceButton）
- 2×2グリッドで画像 + オプションラベル
- 画像はボタン内で `object-cover` 表示
- 最小タッチターゲット 44px 維持

#### 結果画面（ResultsPage）
- 分布バーチャートの左に画像サムネイル（56-80px）を表示
- `sanitizeMediaUrl()` でURL検証済みの画像のみ表示

#### プレビュー画面（PreviewPage）
- 参加者画面と同じレイアウトで画像選択肢をレンダリング
- 正解の選択肢をハイライト表示

### エラーハンドリング
- 画像読み込み失敗時: テキストラベルにフォールバック（ラベル未設定なら「選択肢A」等）
- アップロード失敗時: エラートースト表示（「選択肢Xの画像アップロードに失敗しました」）
- `sanitizeMediaUrl()` が null を返した場合: 画像タグを非表示にしてテキストのみ表示

## F018: ○×問題（2択問題）

> ステータス: 実装済み | 優先度: P1

### 概要
通常の4択に加えて、○（正しい）か×（間違い）の2択で回答する問題形式を追加する。

### 受入基準
- [x] 問題作成時に問題形式を選択: 「4択」「○×」
- [x] ○×問題は正解が「○」か「×」の2択
- [x] 参加者画面: 画面左半分に大きな○ボタン、右半分に大きな×ボタン
- [x] ホスト/プロジェクター画面: ○と×を大きく2つ表示
- [x] 結果画面: ○と×の回答分布を表示
- [x] スコアリングは4択と同じ計算式（速度ボーナスあり）
- [x] 問題バンク（テンプレート）でも○×問題をサポート
- [x] プレビュー画面で○×問題の表示に対応

### DB変更
`questions` と `question_bank` テーブル:
- `question_type` TEXT NOT NULL DEFAULT 'four_choice' — "four_choice" | "true_false"
- ○×問題時: `choice1` = "○", `choice2` = "×", `choice3`/`choice4` = null, `correct_choice` = 1 or 2

### UI

#### 参加者画面（AnswerPage）
- 2ボタンレイアウト（左右分割、`grid-cols-2`）
- ○ボタン: パステルグリーン系、大きな○マーク
- ×ボタン: パステルローズ系、大きな×マーク
- 最小タッチターゲット: 44px以上（実質画面半分なので十分）

#### ホスト/プロジェクター画面
- 2ボタンレイアウトで○と×を大きく表示
- 回答分布も○/×の2本バー

### 影響範囲
- DB: schema.ts (question_type カラム追加)
- Backend: routes/question.ts, routes/questionBank.ts (バリデーション分岐)
- Frontend: QuestionInlineForm, ChoiceButton, AnswerPage, QuestionPage, ResultsPage, PreviewPage

## F019: ポイント倍率問題

> ステータス: 実装済み | 優先度: P1

### 概要
特定の問題のポイントを倍にする「ボーナス問題」機能。事前設定とゲーム中指定の両方に対応。

### 受入基準
- [x] 問題作成時に倍率を設定可能: 1x（デフォルト）, 2x, 3x
- [ ] ゲーム中にホストが出題時に倍率を変更可能
- [x] 倍率適用時、参加者画面に「ボーナス問題！」「ポイント2倍！」等のバナー表示
- [x] ホスト/プロジェクター画面にも倍率バッジ表示
- [x] スコア計算: 通常スコア × 倍率
- [x] ランキングに倍率分のスコアが正しく反映
- [x] ゲーム履歴・統計にも倍率情報を保持

### DB変更
`questions` テーブル:
- `point_multiplier` INTEGER NOT NULL DEFAULT 1 — 1, 2, or 3

### スコアリング
```
finalScore = baseScore * (1 - responseTime / timeLimit * 0.5) * pointMultiplier
```
- 正解: (500〜1000) × 倍率
- 不正解: 0点（倍率に関係なく）

### UI

#### 問題作成（QuestionInlineForm）
- 倍率セレクター: 1x / 2x / 3x（ラジオボタンまたはセグメント）
- 2x以上の場合は「ボーナス問題」バッジ表示

#### ゲーム中のホスト操作
- 出題前に倍率変更ボタン（ドロップダウンまたはトグル）
- 変更はその問題にのみ適用

#### 参加者画面
- 倍率2x以上の問題: 画面上部に「ボーナス問題！ポイントX倍！」バナー
- バナーの色: ゴールド系

#### ホスト/プロジェクター画面
- 問題表示時に倍率バッジ（「×2」「×3」）
- 結果画面でも倍率込みのスコアを表示

### Socket.io
- `questionStarted` ペイロードに `pointMultiplier` フィールドを追加
- `setMultiplier` — ホストがゲーム中に倍率を変更するイベント（任意）

## F020: チーム対戦モード

> ステータス: 実装済み | 優先度: P0

### 概要
参加者をチームに分けて、チーム単位でも競い合えるモード。個人ランキングとチームランキングの両方を表示する。

### 受入基準
- [x] ホストがクイズ設定でチームモードを有効化できる（トグルスイッチ）
- [x] ホストがチーム数とチーム名を事前設定（2〜10チーム）
- [x] 参加者が参加時にチームを選択（ボタン形式）
- [x] チーム選択UIをプロフィール画面に追加
- [x] ゲーム中の回答・スコアリングは個人単位（通常と同じ）
- [x] ランキング画面: チームランキング（上部）+ 個人Top10（下部）の両方を表示
- [x] チームスコア = メンバーの合計スコア
- [x] 最終結果発表: teamReveal（チームランキング）→ scroll → top3 → done → group
- [x] 参加者スマホに自分のチーム順位も表示
- [x] チーム非使用時は従来通りの個人戦のみ（後方互換）
- [x] リプレイ時もチーム割り当て維持（スコアのみリセット）

### DB変更

`quizzes` テーブル:
- `team_mode` INTEGER NOT NULL DEFAULT 0 — チームモードON/OFF

新テーブル `teams`:
- `id` INTEGER PRIMARY KEY
- `quiz_id` INTEGER NOT NULL (FK → quizzes, ON DELETE CASCADE)
- `name` TEXT NOT NULL
- `order_index` INTEGER NOT NULL DEFAULT 0

`participants` テーブル:
- `team_id` INTEGER — nullable (FK → teams, ON DELETE SET NULL)

### チーム管理

#### ホスト設定（QuizConfigTab）
- クイズ設定カード内に「チーム戦モード」トグル（role="switch"）
- ON時: チーム名入力フォーム（最低2、最大10チーム）
- デフォルトチーム名: 「チーム1」「チーム2」（編集可能）
- 「+ チーム追加」ボタン / 「×」で削除
- 「チームを保存」ボタン → PUT `/api/quizzes/:id/teams`

#### 参加者のチーム選択（ProfilePage）
- チームモード有効時: ニックネーム入力の下にチーム選択ボタン一覧
- 選択中のチームは ring で強調表示
- チーム未選択では「参加する」ボタン disabled

### ランキング表示

#### ホスト/プロジェクター画面（RankingPage）
- 上部: チームランキング（チーム名、合計スコア、メンバー数、バーアニメーション）
- 下部: 個人Top10（通常のランキング表示）

## 非機能要件

### パフォーマンス
- 最大チャンク: 250KB以下（manualChunks + lazy load）
- ルートレベル遅延読み込み（Suspense + lazy）
- db.batch() でN+1回避
- ホスト/プロジェクター画面は 1920×1080 に最適化（max-h/max-w制約、フォントサイズ拡大）

### アクセシビリティ
- prefers-reduced-motion 対応
- 44px最小タッチターゲット
- WCAG AA コントラスト比
- aria-label / aria-pressed / aria-live
- セマンティックHTML
- touch-action: manipulation
- overscroll-behavior: contain

### セキュリティ
- watchRoom認証: なし（会場LAN前提）
- 回答二重送信防止: UNIQUE制約（question_id + participant_id）
- host_secret: sessionStorageに保存（URLパラメータには露出しない）
- REST APIは認証なし（host_secretはSocket.ioゲーム操作にのみ使用）
- パストラバーサル対策: メディア配信時にデコード後のパス検証
- ファイルアップロード: IP単位レート制限あり
- ニックネーム: サーバー側で制御文字・特殊Unicode除去
- 参加者一括削除: 配列長上限あり
- メディアURL: スキーム検証（http/https/dataのみ許可）
- Socket.io: 本番環境では websocket-only（pollingなし）
- ファイルクリーンアップ: 手動対応
- データ保持: 手動管理（cascade削除のみ）

### インフラ
- Azure Static Web Apps（Frontend） + Azure App Service B1 Linux（Backend）
- CORS: `CORS_ORIGIN` 環境変数でカンマ区切り指定
- Socket.io: `VITE_API_URL` 環境変数で接続先指定
- CI/CD: GitHub Actions で自動ビルド・デプロイ（**構築予定**）

### 運用制約（決定事項）
- ルームコード: 4桁数字（10,000通り、同時1ゲーム運用で十分）
- 同時ゲーム: 1ゲームのみ
- 動画問題: 実装しない（画像のみ）
- 自撮りフレーム: 固定セット（カスタマイズなし）
- クイズインポート: 不要（エクスポートはF022で対応）
