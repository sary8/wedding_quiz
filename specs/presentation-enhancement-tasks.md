### Task 1: データ構造の準備

**変更ファイル:** `types/index.ts`（frontend/backend 両方）

**内容:**
- `FinalRankingEntry` 型に `averageResponseTimeMs` が含まれていることを確認
- `FinalPhase` の型定義を拡張:
  ```
  既存: "teamReveal" | "scroll" | "top3" | "done" | "group"
  新規: "teamReveal" | "batchScroll" | "top5Wait" | "spotlight" | "done" | "group"
  ```
- `SpotlightData` 型を追加:
  ```ts
  type SpotlightData = {
    rank: number;
    nickname: string;
    avatarUrl: string | null;
    score: number;
    averageResponseTimeMs: number;
    stage: "tease" | "reveal";
  };
  ```

**受入基準:**
- [ ] 型定義が frontend/backend で一致
- [ ] 既存のビルドが通ること

**見積もり:** 30分

---

### Task 2: バッチ分割ロジック

**変更ファイル:** `frontend/src/pages/host/FinalPage.tsx`（新規ユーティリティ関数）

**内容:**
バッチ分割ロジックを実装:
```
入力: rankings（全参加者ランキング配列）
出力: batches[]（各バッチの参加者配列 + メタデータ）

ルール:
- 6位以下: 20人ずつのバッチ（端数は最初のバッチ）
- 10位〜6位: 最後のバッチ（低速スクロール）
- 5位〜1位: バッチではなくスポットライト（別処理）
- 参加者20人以下: 6位以下が1バッチのみ
```

**受入基準:**
- [ ] 100人、50人、20人、10人、5人のケースでテスト
- [ ] ユニットテスト追加
- [ ] バッチごとにスクロール速度（150ms/800ms）が正しく設定される

**見積もり:** 1時間

---

### Task 3: バッチスクロールUI

**変更ファイル:** `frontend/src/pages/host/FinalPage.tsx`

**内容:**
既存の `scroll` フェーズを `batchScroll` フェーズに置換:
- 各バッチの行が下から1行ずつ出現し、上に積み上がる
- 行デザイン: 順位(w-14) | アバター(36px丸) | ニックネーム(flex-1) | スコア(w-24) | 平均回答時間(w-28)
- 偶数行: bg-white/5 ストライプ
- 全行表示後 2秒停止 → フェードアウト → 次バッチ
- バッチ間遷移は自動

**受入基準:**
- [ ] 1位が上、下位が下の並び順
- [ ] 高速バッチ: 1行 150ms
- [ ] 低速バッチ（10-6位）: 1行 800ms
- [ ] 各バッチの最後に順位帯テキスト表示（「21位〜30位」）
- [ ] `prefers-reduced-motion` 時はバッチごとに即時全表示
- [ ] DisplayPage でも同じ表示（`isDisplay` prop）

**見積もり:** 2-3時間

---

### Task 4: Top5待機UI + Socket.ioイベント

**変更ファイル:**
- `frontend/src/pages/host/HostPage.tsx`（ボタン追加）
- `backend/src/socket/quizHandler.ts`（`revealNextRank` ハンドラ）
- `frontend/src/pages/host/FinalPage.tsx`（待機画面）

**内容:**
- バッチスクロール完了後、`top5Wait` フェーズに遷移
- HostPage に「次の順位を発表」ボタンを表示
- ボタン押下 → `revealNextRank` Socket.ioイベント送信
- サーバーは次の順位データを全クライアントに配信
- `revealNextRank` は既存イベント（コードベースに実装済み）を活用

**受入基準:**
- [ ] ホストが5回ボタンを押すことで5位→1位まで発表
- [ ] DisplayPage ではボタン非表示、発表のみ同期
- [ ] 参加者画面にも順位発表が反映

**見積もり:** 1-2時間

---

### Task 5: スポットライト演出 — じらしステージ

**変更ファイル:** `frontend/src/pages/host/FinalPage.tsx`

**内容:**
各順位発表の第1段階:
- 画面中央に「第○位は…」テキスト（text-7xl）
- テキスト微振動（x: [-2, 2], 150ms周期、Framer Motion）
- ドラムロール効果音（既存 `playDrumRoll`）
- 背景色がメダル色/アクセント色へ徐々に変化

じらし時間:
| 順位 | 時間 |
|------|------|
| 5位 | 1.5秒 |
| 4位 | 2秒 |
| 3位 | 2秒 |
| 2位 | 2.5秒 |
| 1位 | 3.5秒 |

**受入基準:**
- [ ] 各順位で正しいじらし時間
- [ ] ドラムロール効果音が再生される
- [ ] `prefers-reduced-motion` 時はテキスト振動なし、背景色は即時変更

**見積もり:** 1-2時間

---

### Task 6: スポットライト演出 — 発表ステージ

**変更ファイル:** `frontend/src/pages/host/FinalPage.tsx`

**内容:**
じらし後の第2段階:
- アバターが scale 0→1 でズームイン（spring, stiffness:120）
- 名前が y:30→0 でスライドアップ（delay:0.3s）
- スコア + 平均回答時間がフェードイン（delay:0.6s）
- ファンファーレ効果音（既存 `playFanfare`）

紙吹雪:
| 順位 | 演出 |
|------|------|
| 5位 | なし |
| 4位 | なし |
| 3位 | ブロンズ色 50個 |
| 2位 | シルバー色 100個 |
| 1位 | ゴールド色 200個 × 3回 + 追加2回（計5回） |

1位追加演出:
- 画面フラッシュ（白→透明, 200ms）
- 背景全体が金色グラデーションにアニメーション変化

表示レイアウト:
```
順位（大きく） → アバター（192px） → 名前 → スコア → 平均回答時間
```

**受入基準:**
- [ ] 各順位で正しい紙吹雪数・色
- [ ] 1位の追加演出（フラッシュ + 金色背景）
- [ ] ファンファーレがランク別に異なる（既存実装を活用）
- [ ] `prefers-reduced-motion` 時はアニメーションなし、即時表示
- [ ] DisplayPage でも完全に同じ演出

**見積もり:** 2-3時間

## リスク・注意事項

- **Framer Motion**: 既存で使用中のため追加依存なし
- **紙吹雪**: 既存実装（`confetti` 関数）を拡張。大量パーティクル（200個×5回）のパフォーマンスを確認
- **DisplayPage同期**: HostPageのフェーズ遷移がSocket.io経由でDisplayPageに正しく伝播することを確認
- **既存の `revealNextRank`**: コードベースに既に実装されているため、活用・拡張する方針
