## 2. 明確化結果（全10件 解決済み）

### NC-1: BGMファイルの提供方法 → ユーザー任意配置
- `public/audio/` に好きなMP3を配置する方式を維持
- READMEにファイル名規則（`lobby.mp3`, `question.mp3`, `results.mp3`）を記載
- デフォルトBGMは同梱しない

### NC-2: CI/CDパイプライン → GitHub Actions構築
- GitHub Actions でビルド・デプロイを自動化する
- **実装タスクとして追加**

### NC-3: 同時ゲーム運用 → 不要（1ゲームずつ）
- 結婚式は1回、同時開催の必要なし
- 技術的には可能だが、テスト・ドキュメント追加は不要

### NC-4: 動画問題UI → 不要（画像のみで十分）
- 動画アップロードのバックエンドインフラは残すが、フロントUI追加は不要
- 将来必要になれば `<video>` タグ追加で対応可能

### NC-5: ファイルクリーンアップ → 手動対応
- `uploads/` は必要時に手動クリア
- 結婚式1回のイベントなので自動削除は不要

### NC-6: データ保持ポリシー → 手動管理
- NC-5 と同様、クイズ削除（cascade）で対応
- 自動期限切れ機能は実装しない

### NC-7: watchRoom認証 → 現状維持
- 会場LAN内での利用が前提
- ルームコード6文字で十分

### NC-8: レート制限 → 現状維持
- 100人規模の結婚式二次会では不要
- 二重送信防止（UNIQUE制約）は実装済み

### NC-9: 自撮りフレームカスタマイズ → 不要
- 現状の固定フレームセットで十分

### NC-10: クイズインポート/エクスポート → 不要（変更なし）
- 仕様レビュー時の決定を維持
